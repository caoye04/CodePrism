================================================================================
CODE SPECIFICATIONS: RULES vs DIRECTIVES
代码规范：规则 vs 指令
================================================================================

You can find in our "code_specifications_list" two types of coding standards:Rules and Directives.
在我们的"code_specifications_list"中，您可以找到两类代码标准：规则和指令。

================================================================================

Rules are coding standards that can be fully verified by automated static analysis tools. They have objective, binary pass/fail criteria.
规则是可以通过自动化静态分析工具完全验证的编码标准。它们具有客观的、二元的通过/不通过标准。

Examples | 示例:
- Syntax violations (语法违规)
- Type mismatches (类型不匹配)
- Unused code detection (未使用代码检测)

================================================================================

Directives are coding standards that cannot be fully automated. They require: Human judgment、Project context understanding、Documentation review、Requirements traceability
指令是无法完全自动化的编码标准。它们需要人工判断、项目上下文理解、文档审查和需求追溯。

Examples | 示例:
- Code-to-requirement traceability (代码到需求的可追溯性)
- Implementation-defined behavior documentation (实现定义行为的文档化)
- Error handling verification (错误处理验证)

================================================================================

WHY AI MATTERS? | 为什么AI很重要？

Traditional static analysis tools can only check Rules automatically.Directives have always required manual code review, which is:
传统静态分析工具只能自动检查规则。指令一直需要人工代码审查，这存在以下问题：

✗ Time-consuming and expensive (耗时且成本高)
✗ Inconsistent across reviewers (不同审查员标准不一致)
✗ Difficult to scale (难以扩展)
✗ Prone to human oversight (容易出现人为疏漏)

AI-POWERED CODE REVIEW can bridge this gap by:
AI驱动的代码审查可以填补这一空白：

✓ Understanding code context and intent (理解代码上下文和意图)
✓ Checking documentation completeness (检查文档完整性)
✓ Verifying requirements traceability (验证需求可追溯性)
✓ Identifying semantic issues (识别语义问题)
✓ Providing consistent, scalable reviews (提供一致的、可扩展的审查)

AI enables semi-automated verification of Directives that were previously impossible to check automatically, significantly improving code quality and reducing manual review burden.

AI使得过去无法自动检查的指令能够实现半自动化验证，显著提高代码质量并减轻人工审查负担。
